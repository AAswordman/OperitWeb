# 小白教程：AI 提供商、Token、请求地址与模型是什么？

如果你是第一次配置 AI，看到“提供商”“Token”“请求地址”“模型”这些词很容易发懵，其实可以把它想象成一次点外卖。你先决定去哪家店下单，这家店就是 AI 提供商；你要用自己的账号去结算，账号对应的身份凭证就是 Token；你还要把订单发到正确的门店地址，这就是请求地址；最后你会选一个具体菜品，决定口味和价格，这个就是模型。

先说 AI 提供商。它本质上是一家提供大模型服务的平台，比如 OpenAI、Anthropic、Google 或者其他兼容 OpenAI 接口的平台。Operit AI 本身不替你“生产”模型，而是把你的问题转发到你选择的提供商，再把结果返回给你，所以你需要先在提供商平台开通账号并创建可用的 API 凭证。

Token 在很多平台里也叫 API Key，可以把它理解为“你是谁、费用记在谁名下”的钥匙。应用在发请求时会携带这个钥匙，平台据此判断权限、统计用量并计费。它非常敏感，谁拿到它就可能用你的额度，所以不要随便发给别人，也不要公开到代码仓库或截图里。一般只要在应用里配置一次，后续正常使用就不需要反复填写。

请求地址通常是一个 URL，也就是接口入口地址。你可以把它看成“把请求送到哪一扇门”。如果地址写错、协议不兼容，哪怕 Token 正确也会报错。很多官方平台会给出默认地址，一些中转或聚合平台也会提供自己的兼容地址。对于新手来说，最稳妥的做法是先用提供商文档给出的标准地址，确认跑通后再做个性化调整。

模型则决定了 AI 的能力、速度和成本。通俗一点说，模型就是不同档位的“脑力引擎”：有的更快更便宜，适合日常问答和轻量任务；有的更强更贵，适合复杂推理、长文本处理或高质量生成。你在应用里选择的模型名称，本质上就是告诉提供商“这次要调用哪台引擎”。如果同样的问题换一个模型，回答质量、响应速度和花费都可能明显不同。

很多人会问：为什么要付费？更常见的疑问是，为什么一些官方 App 看起来是免费的。这里的关键是“看起来免费”不等于“没有成本”。大模型每次回答都在消耗真实算力，背后是昂贵的 GPU、带宽、电力、存储和运维投入，所以平台通常会用产品策略来分摊成本，比如给免费层设置次数限制、速度限制、可用模型限制，或者用会员订阅、生态补贴和商业转化来覆盖开销。也就是说，官方 App 的免费体验往往是一个入口，让你先用起来，而不是无限、无门槛、长期稳定地提供最高规格服务。

那为什么很多进阶用户还是更愿意用 API？因为 API 调用的好处是可控、可配置、可集成。你可以自己选模型、切换提供商、控制温度和上下文长度，能把 AI 接进自己的工作流、自动化脚本和应用里，也能更清楚地看到每次调用的消耗与成本，按需优化效果和预算。对于需要稳定产出、批量处理、或者希望把 AI 能力嵌入产品的人来说，API 不是“更麻烦的入口”，而是“更专业的控制台”。

如果你刚开始使用，建议先选择官方推荐的入门模型，保持默认参数，用短问题做几次测试，确认“能通、能回、速度可接受”后再逐步升级模型。这样你会更快建立对价格和效果的直觉，也能避免一上来就因为配置复杂或成本失控而挫败。
