### 制作Web：使用工作区

想在 Operit 里实现“写网页 → 预览 → 打包”一条龙，第一步是选对工作区类型。

![设置工作区](/manuals/assets/workspace/image.png)

#### 先选对工作区（可直接打包的关键）

- **可直接打包网页：** `工作区 > 创建默认 > Web 项目`
- 该模板会自动创建 `index.html`，并在 `.operit/config.json` 中默认启用 `export.enabled=true`。
- 其他默认模板（Android / Node.js / TypeScript / Python / Java / Go / 办公文档 / 空白）默认 `export.enabled=false`，不会直接显示导出按钮。
- `附加本地储存仓库`（SAF）当前用于外部目录协作，聊天工作区里不显示导出按钮。

#### 三种工作区入口（按 UI 文案）

- **创建默认**：在应用内部创建新工作区，适合新项目。
- **选择现有**：进入内置文件浏览器，定位目录后点击 `绑定当前文件夹`。
- **附加本地储存仓库**：通过系统文件夹选择器绑定 SAF 目录。

#### 绑定 SAF 工作区（用户视角）

1. 在聊天页打开工作区面板，点击 `附加本地储存仓库`。
2. 在系统文件夹选择器中选中目标目录并授权。
3. 弹出 `仓库名称` 对话框，填写 `名称`（名称不能为空且不能重复）。
4. 点击确定后即可绑定；后续 AI 会在该仓库目录中读写文件。

> 提示：SAF 绑定后，工作区会使用仓库环境（例如 `repo:你的名称`）访问文件，适合跨目录协作。

#### 工作区交互能力

- AI 可以读取、创建、修改、删除文件。
- 文件列表会遵循 `.gitignore` 规则过滤（例如自动忽略 `node_modules`）。
- 需要回退文本修改时，可长按你自己的消息并选择 `修改并重发`。
