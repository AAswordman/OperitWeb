# 12. 工作区基础

前面把工具相关的东西都讲了一遍，这一章来说说另一个重要的东西——工作区。

## 什么是工作区

工作区这个概念其实很简单，就是给当前对话**绑定一个文件夹**，让 AI 能在这个文件夹里读写文件。

没有工作区的时候，AI 也能创建文件和读取文件，但文件会放得乱七八糟，没有固定的目录结构。有了工作区，AI 就知道该在哪个文件夹里操作，文件不会到处乱放——你可以把它理解为 AI 的*办公桌*，它就在这张桌子上干活。

在之前的章节（第7章）我们也提到过，工作区的内容会作为*提示词*的一部分注入到上下文中，AI 知道当前工作区的结构和文件内容，这样才能有针对性地操作。

## 绑定工作区

在工作区的入口，你会看到两个选项。

![设置工作区](</manuals/assets/workspace/setup_workspace.jpg>)

### 创建默认

这是最省事的办法，适合从头开始的新项目。点「**创建默认**」，应用就会在内部帮你创建一个独立的工作区，然后弹出模板选择界面让你选一个项目模板。

![选择工作区模板](</manuals/assets/workspace/template_selector.jpg>)

模板有空白工作区、办公文档、Web 项目、Android 项目、Flutter 项目、Node.js 项目等等可选，每种模板对应不同类型的项目，带着不同的*预设配置*。比如 Web 项目默认就有 `index.html` 和导出入口，Android 项目会带上 Gradle 快捷命令。不确定选什么的话，办公文档和 Web 项目这两个最通用。

选了不同模板之后，工作区的界面也会不一样。Web 项目模板默认显示的是浏览器预览界面，能看到网页效果；而 Android 项目模板显示的是一排快捷命令按钮，比如初始化环境、构建 APK、运行测试这些。其他项目模板也类似，每个模板的界面都是根据它的用途定制的。

### 选择现有

如果你已经有一个项目文件夹，想直接交给 AI 来处理，就选这个。点进去之后会打开内置的文件浏览器，找到你的文件夹，点「**绑定当前文件夹**」就行了。适合导入已有的项目让 AI 继续开发或者做修改。

需要注意的是，选择现有文件夹不会自动创建 `.operit` 配置目录，所以预览之类的功能可能会报找不到配置。如果你需要这些功能，可以参考模板里的配置文件手动放一个到工作区根目录下。

![内置文件浏览器](</manuals/assets/workspace/file_browser.jpg>)

在文件浏览器的顶部有几个标签页（Linux、SDCard、Workspace），右侧还有一个 **+ 号按钮**。点 + 号可以通过 Android 系统的 *SAF（Storage Access Framework）*来选择外部目录并绑定为*仓库*，比如访问 Termux 的内部存储。绑定时会给仓库起个名字，之后 AI 就在这个仓库环境里读写文件。这种方式适合访问常规路径无法直接到达的目录。

三种方式选哪个取决于你的场景：新项目用"创建默认"，已有项目用"选择现有"，需要跨目录协作或者操作外部存储就进文件浏览器点 + 号绑定仓库。

## 工作区里有什么

工作区绑定好之后，AI 就可以在里面做很多事情了——创建文件、编辑代码、删除不需要的东西。如果你用的是 Web 项目模板，还可以直接在工作区里预览网页效果，这个预览是解除了 CORS 限制的，不用担心跨域问题。预览服务默认开在 8093 端口，你也可以在手机其他浏览器里访问 `http://127.0.0.1:8093` 来看效果。如果需要让 AI 做自动化 Web 开发，可以让它启用自动化浏览器工具包去打开这个地址进行调试。工作区还支持**重命名**和**解绑**。**解绑不会删除已有的文件**，文件都会保留在原地。不过需要注意的是，如果你解绑之后又点「创建默认」新建一个工作区，**它不会覆盖旧的那个**，而是创建一个新的。所以如果旧的工作区还需要复用，建议**新开一个对话**来绑定。

另外工作区还有一个*回档*功能：当你在对话里回退历史的时候，工作区里的文件也会跟着恢复到当时的状态。虽然不一定每次都*百分之百靠谱*，但有总比没有好。工作区的 `.backup` 目录里存的就是这些备份文件，回档就是靠它们来实现的。

![工作区命令面板](</manuals/assets/workspace/workspace_dashboard.jpg>)

每个工作区根目录下会有一个隐藏的 `.operit/config.json` 文件，这个文件记录着工作区的配置。包括*项目类型*、*预览设置*、*快捷命令按钮*等等。如果你对配置熟悉，可以自己改这个文件来调整工作区的行为，不过对新手来说，直接用模板生成的配置就够了。

工作区还支持一个特殊的 `AGENTS.md` 文件——如果你在工作区根目录下放了这个文件，每次发送消息的时候，它的内容会自动追加到系统提示词里。这个文件很适合写一些你希望 AI 在整个对话过程中都记住的行为规范或者项目背景。

另外，如果你想在文件管理器（比如 MT 管理器）里直接操作工作区的文件，可以通过「添加本地储存」把工作区目录加进去。这样就能像浏览普通文件夹一样看到工作区里的内容了。

![MT管理器中的工作区](</manuals/assets/workspace/mt_manager_workspace.jpg>)
![添加本地储存](</manuals/assets/workspace/mt_add_storage.jpg>)

## 导出

如果你是用"创建默认"并且选了 Web 项目模板做出来的东西，工作区右下角会有一个导出按钮。点它就可以把你的网页打包成 Android 应用（*APK*）或者 Windows 应用（*EXE*）。点击导出后可以选平台、填包名、应用名、版本、图标等信息，完成后产物默认保存在 `Download/Operit/exports/` 目录下。

![导出打包](</manuals/assets/teach_step/1-1.png>)
![选择平台](</manuals/assets/teach_step/1-2.png>)
![填写应用信息](</manuals/assets/teach_step/1-3.jpg>)

需要注意的是，**只有 Web 项目模板才支持导出**。其他类型的工作区没有导出按钮——比如 Android 项目模板，你得手动点编译按钮构建 APK，然后通过分享功能把文件发出去；办公文档的话，就让 AI 帮你生成文件后分享出来。
