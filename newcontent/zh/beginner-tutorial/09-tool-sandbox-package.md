# 09. 工具：沙盒包

这是软件特有的一种工具形式，比skill能力大，比mcp轻量级。
承接我们上一节关于渐进式工具加载的内容继续。沙盒包是以quickjs引擎，配合软件预制的函数运行的一种js脚本。目前它可以使用Tools.*调用软件的一切内置工具，也可以使用JavaBridge直接交互和触碰软件的Kotlin代码，非常的强悍。

软件有很多的内置沙盒包，实现了各种各样的功能。这一节内容，将会详细解析沙盒包的构造和原理，以及软件目前的所有沙盒包。（包内工具就不介绍了，这个在软件里面也是能看到的）

## 内置与外置

沙盒包分两种：内置的和外置的。

内置包就是软件自带的那一批，也就是上面表格里列出来的那些。它们跟着应用走，你不需要额外安装。内置包不支持删除，只能开关——不想用哪个关掉就行。

外置包则是你自己导进去的。导入方式很简单，把 `.js` 脚本文件或者 `.toolpkg` 插件包放到外部 packages 目录（`Android/data/com.ai.assistance.operit/files/packages`）就行了，软件会自动识别。不想用了也可以直接按文件路径删掉。

有意思的是，不管内置还是外置，在 AI 看来完全没有区别。它只需要通过 `use_package` 激活，然后用 `package_proxy` 调用里面的工具就行了，根本不需要知道这个包是哪来的。

## 结构

前面说了，沙盒包本质上就是在 QuickJS 引擎里跑的一段 JS 脚本。但它不是随便写的脚本，而是通过软件预制的函数和接口来和系统交互。

一个普通沙盒包就是一个 `.js` 文件。脚本里可以通过 `Tools.*` 来调用软件的所有内置工具——也就是说，你在08章看到的 read_file、apply_file、visit_web 这些，沙盒包里全都能用。不仅如此，它还能通过 JavaBridge 直接和软件的 Kotlin 代码交互，能力非常强。

在普通沙盒包的基础上，我们还拓展了一种叫 **ToolPkg** 的插件格式。你可以把它理解成一个打包好的插件包，里面可以包含多个脚本、资源文件和配置清单。有点像 Minecraft 的 modpkg 那种感觉——把一堆东西打包在一起，安装一个就能用一套。关于 ToolPkg 的详细制作和使用，会放到后面的教程专门讲。

如果你想自己动手写沙盒包，可以参考官方的脚本开发文档（在 GitHub 上可以获取到），里面有完整的接口说明和示例。

## 应用

光说原理可能有点抽象，下面通过几个实际的应用场景来感受一下沙盒包到底能干什么。

### 绘图输出

你可能会好奇，AI 怎么画图？其实绘图功能就是通过沙盒包来实现的。软件内置了多个绘图相关的沙盒包，每个包对接了不同的绘图服务，比如 MiniMax、OpenAI、硅基流动、通义千问、xAI、智谱等等。

![绘图包列表](</manuals/assets/tools/draw_package_list.jpg>)

你只需要在包管理里启用其中一个可用的绘图包就行了，不需要全部都开。AI 在需要画图的时候，会自动激活对应的包，调用里面的工具来完成绘图，然后把结果输出给你。

不过有一点要注意：绘图包和一般的沙盒包不太一样，它通常需要配置**环境变量**（比如 API Key）。在包管理界面里，每个包的右边有几个按钮，第一个按钮就是"管理配置"弹窗，点进去之后就能看到这个包需要你填写哪些环境变量。比如 MiniMax 的绘图包，就需要你填 `MINIMAX_API_KEY`，这样才能正常调用接口。

![绘图包环境变量配置](</manuals/assets/tools/draw_package_env_config.jpg>)

### 延时提醒

"帮我设个明天早上8点的闹钟"或者"下午3点提醒我开会"——这类日常需求，背后就是 `daily_life` 这个沙盒包在干活。它把系统的闹钟、提醒这些能力封装成了 AI 可以直接调用的工具。AI 收到你的指令后，会自动激活这个包，然后调用里面的提醒或闹钟工具来完成操作。整个过程你只需要说一句话，剩下的都交给 AI 和沙盒包来处理。

### 转换文件

你拿到一个 `.webp` 的图片想转成 `.jpg`，或者一段 `.wav` 的音频想转成 `.mp3`，这些格式转换的操作由 `file_converter` 这个沙盒包来负责。它封装了 FFmpeg 和其他转换工具的能力，支持音频、视频、图片和文档之间的各种格式互转。你只需要告诉 AI 你想把什么文件转成什么格式，它就会激活这个包帮你搞定。
